Gitでバージョン管理

ファイルを共有する。

  1. 「git init」
  2. 対象のディレクトリに移動する。
  3. ファイルを変更して、共有する。「git add ファイル名」「git commit -m "メッセージ"」
  4. リモートリポジトリ（共有ファイルの置き場）の準備。
  5. リモートリポジトリの登録。 remote add リモートリポジトリ名(origin) URL」
  6. リモートリポジトリにファイルをアップロード（プッシュ）する。「git push origin master」
  7. ファイルをダウンロード（プル）する。「git pull origin master」

共同開発で大切なこと。把握・選択

  1. 自分がどのファイルを変更したか把握する。「git status」。変更したファイル名は赤くなる。
  2. 自分が変更した部分の内容を把握する。「git giff」。変更前のコードが赤。変更後のコードが緑。
  3. addしたファイルを確認する。「git status」addされたファイルは緑。addされてないファイルは赤。

変更内容の確認

  • 修正内容のコミットメッセージはわかりやすく書く。「git commit -m "サブタイトルの色の修正"」とか
  • コミットメッセージを確認する。「git log」。
  • 変更内容を確認する。「git log -p」
  • 上下キーで表示範囲の変更。Qキーで終了。

GitHubのリポジトリのアクセス

  • リポジトリにアクセスするには、アクセストークンが必要。
  • GitHub「setting」「Developer setting」「Personal access tokens」「Generate new token」で発行する。
  • 「git clone https://${アクセストークン}@${ユーザー名}/${リポジトリ名}.git」

    ただしこの方法でcloneしたリポジトリでNode.jsの npm init などを行って package.json を生成すると、リポジトリのURLにトークンが付与されてしまうので注意。そのままコミットするとトークンが漏洩するので除去すること。 リモートのURLはコマンド git remote set-url origin ${URL} で変更できる。変更結果はコマンド git remote -v で確認できる。
